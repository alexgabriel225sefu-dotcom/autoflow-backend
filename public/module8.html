Build · Lesson 1

Build a Mini-CRM in Google Sheets

You do not need HubSpot or Salesforce for your first clients. A well-structured Google Sheet does the same job at zero cost — and you can sell the setup for $300–$500. This lesson shows you the exact structure and how to build it in 20 minutes.

Why Google Sheets CRM Works
Most small businesses have zero system for tracking leads. They rely on WhatsApp chats, Instagram DMs, scattered emails, and memory. Leads fall through the cracks every day. A Google Sheets CRM that automatically captures every lead from every source and shows them in one place is a massive transformation for a small business — and it costs you nothing extra to build since you already know Make.com and GPT-4o.
The client opens one Google Sheet every morning and sees every lead from the last 24 hours — who they are, where they came from, what they said, and what the AI recommends as the next action. This alone is worth $300–$500 setup plus $100–$150/month.
Step 1 — Create the CRM Google Sheet
1
Go to sheets.google.com and create a new spreadsheet
Open sheets.google.com. Click the large + to create a blank spreadsheet. Click on "Untitled spreadsheet" at the top left and rename it: [Client Name] — Lead CRM.
2
Set up the column headers in Row 1
Click on cell A1 and type the first column header. Press Tab to move to the next cell. Enter all these headers across Row 1:
ColumnHeader NameHow it gets filled
ATimestamp✦ Automatic — Make.com fills this
BLead Name✦ Automatic — from form/message
CEmail✦ Automatic — from form
DPhone✦ Automatic — from WhatsApp
ESource✦ Automatic — Form/WhatsApp/Instagram/Email
FMessage✦ Automatic — what they said
GStatusManual — New/Contacted/Call Scheduled/Client/Lost
HAI Summary✦ Automatic — GPT-4o generates this
INext Action✦ Automatic — GPT-4o suggests this
JNotesManual — client writes their own notes
3
Format the headers row
Select the entire Row 1 by clicking the "1" on the left side. Then: click Format in the top menu → Bold (or press Ctrl+B). Change the background color by clicking the paint bucket icon in the toolbar → choose a dark green or any color that contrasts with white text. This makes the headers stand out clearly.
4
Freeze Row 1 so headers always stay visible
Click View in the top menu → Freeze → 1 row. Now when the client scrolls down through hundreds of leads, the column headers always remain visible at the top.
5
Add a Status dropdown validation to Column G
Click on cell G2. Hold Shift and click G1000 to select the entire G column from row 2 to 1000. Then click Data in the top menu → Data validation. In the sidebar that appears, click Add rule.
In the rule settings:
— Criteria: select Dropdown
— Add these options one by one: New, Contacted, Call Scheduled, Client, Lost
— Click Done

Now every cell in column G has a dropdown menu. The client clicks a cell and selects the lead status from the list — no typing needed, no mistakes.
6
Add conditional color formatting to Status column
Select column G (rows 2 to 1000 as before). Click Format → Conditional formatting. Add rules to color each status:
— "New" → light blue background
— "Contacted" → yellow background
— "Call Scheduled" → orange background
— "Client" → green background
— "Lost" → red background

For each rule: click Add another rule → under "Format cells if" select Text is exactly → type the status name → click the fill color square and choose the color → click Done.

Now the Status column is color-coded — the client can see at a glance how many leads are in each stage.
7
Share the sheet with the client
Click the Share button in the top right. Enter the client's email address. Set their permission to Editor so they can update the Status and Notes columns. Click Send. The client receives an email with a link to the CRM.

Integration · Lesson 2

Connect All Lead Sources — Form + WhatsApp + Instagram + Email

The power of the CRM comes from having every lead source flow into the same sheet automatically. This lesson connects Google Forms, WhatsApp (360dialog), and Instagram to the CRM — so no lead is ever missed.

The Architecture — One CRM, All Sources
You already built three separate automations in Modules 2, 5, and 6. Each of them handles a different channel. Now you add one extra step to each of those Make.com scenarios: after handling the message (sending the reply), add a Google Sheets "Add a Row" module that logs the lead into the CRM.
You also add a GPT-4o step that generates a 2-line summary of the lead and a suggested next action — saving the client from having to think about what to do with each lead.
Source 1 — Connect Google Form to CRM
1
Open the lead follow-up scenario from Module 2
Go to Make.com → Scenarios. Open the "Lead Follow-Up" scenario you built in Module 2. This scenario already has: Google Sheets (trigger) → OpenAI → Gmail. You will add two more modules at the end.
2
Add OpenAI to generate the AI Summary
After the Gmail module, click +. Add OpenAI → Create a Chat Completion. Use the same connection (your API key). Model: gpt-4o. In the Message content field paste this prompt with variables:
Summarize this lead in maximum 2 sentences and suggest one specific next action. Lead name: {{1.Full_Name}} Lead email: {{1.Email_Address}} Their message: {{1.What_is_your_main_problem_or_question}} Source: Google Form Format your response as: Summary: [2 sentence summary] Next Action: [one specific recommended action]
3
Add Google Sheets "Add a Row" to log the lead in the CRM
After the OpenAI summary module, click +. Add Google Sheets → Add a Row. Connect the same Google account. Select the CRM spreadsheet you built in Lesson 1. Map each column:
— Column A (Timestamp): Click variable picker → select the built-in function now() or type {{now}}
— Column B (Lead Name): Variable → Google Sheets module → Full Name field
— Column C (Email): Variable → Google Sheets module → Email Address field
— Column D (Phone): Leave empty — Google Forms usually don't capture phone
— Column E (Source): Type Google Form directly — this is always the same for this scenario
— Column F (Message): Variable → Google Sheets module → What is your main problem field
— Column G (Status): Type New — all new leads start as New
— Column H (AI Summary): Variable → last OpenAI module → choices[] → message → content
— Column I (Next Action): Same variable as H — the GPT output contains both summary and next action
— Column J (Notes): Leave empty — the client fills this manually

Click OK. Save the scenario.
Source 2 — Connect WhatsApp Bot to CRM
1
Open the WhatsApp Bot scenario from Module 5
Go to Make.com → Scenarios. Open the WhatsApp Bot scenario. It currently has: Webhooks → OpenAI → HTTP (send reply). You will add the same two modules at the end: a new OpenAI for the summary, and Google Sheets to log the lead.
2
Add OpenAI for the CRM summary after the HTTP module
Click + after the HTTP module. Add OpenAI → Create a Chat Completion. Paste this prompt:
Summarize this WhatsApp lead in maximum 2 sentences and suggest one specific next action. Sender phone number: {{1.messages[].from}} Their message: {{1.messages[].text.body}} Source: WhatsApp Format: Summary: [2 sentence summary of what they want] Next Action: [one specific recommended action]
3
Add Google Sheets "Add a Row" to log the WhatsApp lead
Click +. Add Google Sheets → Add a Row. Select the same CRM sheet. Map the columns:
— Column A (Timestamp): {{now}}
— Column B (Lead Name): Leave empty — WhatsApp doesn't always provide a name. Type "WhatsApp Contact" or leave blank.
— Column C (Email): Leave empty
— Column D (Phone): Variable → Webhooks module → messages[] → from (this is their WhatsApp number)
— Column E (Source): Type WhatsApp
— Column F (Message): Variable → Webhooks module → messages[] → text → body
— Column G (Status): Type New
— Column H (AI Summary): Variable → new OpenAI module → choices[] → message → content
— Column I (Next Action): Same
— Column J (Notes): Leave empty

Click OK. Save the scenario.
Source 3 — Connect Instagram DM Bot to CRM
1
Open the Instagram DM Bot scenario from Module 6
Go to Make.com → Scenarios. Open the Instagram DM Bot. It has: Webhooks → OpenAI → HTTP (send reply). Add the same two modules at the end.
2
Add OpenAI for the CRM summary
Click + after the HTTP module. Add OpenAI → Create a Chat Completion. Paste:
Summarize this Instagram DM lead in maximum 2 sentences and suggest one specific next action. Sender Instagram ID: {{1.entry[].messaging[].sender.id}} Their message: {{1.entry[].messaging[].message.text}} Source: Instagram DM Format: Summary: [2 sentence summary] Next Action: [one specific recommended action]
3
Add Google Sheets "Add a Row"
Click +. Add Google Sheets → Add a Row. Select the CRM sheet. Map:
— Column A: {{now}}
— Column B: Leave empty or type "Instagram User"
— Column C: Leave empty
— Column D: Leave empty
— Column E: Type Instagram DM
— Column F: Variable → message text from webhook
— Column G: Type New
— Column H: Variable → OpenAI output
— Column I: Same
— Column J: Leave empty

Click OK. Save the scenario.

Automation · Lesson 3

Automated Weekly Report — AI Writes the Client Summary

Every Monday morning the client receives an email with a complete summary of the previous week's leads — automatically written by GPT-4o. This takes zero time from you once built, dramatically increases client retention, and justifies the monthly retainer forever.

Why the Weekly Report is Your Best Retention Tool
Clients cancel when they forget what value they are receiving. The weekly report makes the value visible every Monday. When a client sees "You received 23 leads this week — 8 from WhatsApp, 11 from Instagram, 4 from the contact form. Your fastest-growing source is Instagram DMs up 40% from last week" — they never think about cancelling. The report is proof the system is working.
Step 1 — Create the Weekly Report Scenario in Make.com
1
Create a new Scenario in Make.com
In Make.com, click + Create a new scenario. Click the trigger circle. Instead of choosing an app, look for the Clock icon or search for Scheduling. This is Make.com's built-in time-based trigger. Select it.
The Scheduling trigger runs the scenario at a specific time on a recurring schedule — without needing any external event to trigger it. This is different from the Google Sheets "Watch New Rows" trigger which needs data to arrive.
2
Configure the schedule — Every Monday at 9:00 AM
In the Scheduling settings, configure:
— Run scenario: Select Weekly
— Day of the week: Select Monday
— Time: Set to 09:00
— Timezone: Select the client's local timezone

Click OK. The scenario will now run automatically every Monday at 9am.
Step 2 — Read Last Week's Data from the CRM Sheet
1
Add Google Sheets "Search Rows" module
Click + after the Schedule trigger. Search Google Sheets and select Search Rows (not Watch New Rows — you want to read existing data, not watch for new ones).
Connect your Google account. Select the client's CRM spreadsheet. Select the correct sheet tab.
2
Filter for rows from the last 7 days
In the Search Rows settings, find the Filter field. Set up a filter to only return rows where the Timestamp (Column A) is from the last 7 days.
Click Add filter rule:
— Column: A (Timestamp)
— Condition: Greater than or equal to
— Value: Use the Make.com formula function. Click the variable picker and look for date functions. Type: {{addDays(now; -7)}}

This formula means "7 days ago from right now." Make.com will only return rows where the Timestamp is from the last 7 days.

Set the Limit to 200 — returns up to 200 rows from the last week.
Step 3 — Aggregate the Data with Array Aggregator
1
Add a Text Aggregator to combine all rows into one text block
The Search Rows module returns multiple rows — one bundle per row. But GPT-4o needs all the data as one single text input. You need to combine all rows into one string first.
Click +. Search for Text aggregator — it is in the Tools category. Select it.

In the Text aggregator settings:
— Source Module: Select the Google Sheets Search Rows module
— Text: Type this template using variable picker for each value:
Lead: {{2.B}} | Source: {{2.E}} | Message: {{2.F}} | Status: {{2.G}}
— Row separator: Type a line break (press Enter once in the field)

The Text aggregator runs once for each row from Google Sheets and combines all of them into one big text block. Click OK.
Step 4 — Generate the Report with GPT-4o
1
Add OpenAI to write the weekly report
Click +. Add OpenAI → Create a Chat Completion. Model: gpt-4o. In the Message content field paste this prompt, inserting the Text aggregator output as the variable:
You are writing a professional weekly performance report for a business owner. Here is the lead data from the past 7 days: {{3.text}} Write a clear, friendly weekly report in under 200 words that includes: 1. Total number of leads received this week 2. Breakdown by source (how many from each channel) 3. Most common type of inquiry 4. One specific highlight or positive trend 5. One recommendation for next week Use a professional but warm tone. Start with "Good morning [Name]," and end with "Have a great week!" Do not use bullet points — write in natural paragraphs.
Step 5 — Send the Report via Gmail
1
Add Gmail "Send an Email" module
Click +. Add Gmail → Send an Email. Connect the sending Gmail account. Configure:
— To: Type the client's email address directly (or use a variable from a settings sheet if you want to make this reusable)
— Subject:
Your Weekly Lead Report — {{formatDate(now; "DD MMMM YYYY")}}
— Content: Variable picker → OpenAI module → choices[] → message → content

Click OK.
2
Turn on the Scenario
Click the Scheduling toggle at the bottom left of the scenario editor to turn it ON. Click Save. The scenario will now run automatically every Monday at 9am and send the client a professionally written AI-generated weekly report.
3
Test it manually with "Run once"
Click Run once to test right now. Check the client's inbox — they should receive the weekly report email within 30 seconds. Read the report — does it sound natural? Does it reference real data from the CRM? Adjust the GPT prompt if needed until you are happy with the output.
