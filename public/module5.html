WhatsApp Business API — What It Is and How to Get It

WhatsApp has over 2 billion active users. Businesses that reply instantly convert 3–5x more leads than those that reply in hours. To build an automated WhatsApp bot you need API access — this lesson shows you exactly how to get it.

Why WhatsApp API and Not Just WhatsApp Business App
There are two different things called "WhatsApp Business." The first is the free app you download on your phone — it lets you reply manually and set away messages, but it cannot be automated. The second is the WhatsApp Business API — this is a programmatic interface that allows Make.com to receive and send messages automatically.
To build a real bot that replies 24/7 without human involvement, you need the API. You access it through a provider like 360dialog — they connect your phone number to the API and give you a key that Make.com uses to send and receive messages.
The Three API Options — Which One to Use
1
360dialog — Recommended for beginners
360dialog is the cheapest and simplest option. It costs approximately $5–$10/month per phone number and integrates directly with Make.com. Setup takes about 30 minutes. This is what we use in this module.
Website: 360dialog.com — you create an account, connect a phone number, and get an API key. Make.com has a built-in 360dialog integration so no complex configuration needed.
2
Twilio — More stable, more expensive
Twilio is a larger platform that also provides WhatsApp API. It costs more ($15–$20/month) but is more reliable for high message volumes. Use Twilio when a client sends or receives more than 500 messages per day.
Website: twilio.com — the setup is similar to 360dialog but more steps. The Make.com integration uses the HTTP module with Twilio's API endpoint.
3
Meta Business API — Free but complex
Meta offers the WhatsApp Business API directly for free. There is no monthly fee but the setup requires creating a Meta Business account, a Facebook App, and going through a verification process that can take days. Use this option only once you are comfortable with the simpler options.
Website: developers.facebook.com — for your first bots use 360dialog and upgrade clients to Meta API later if they want to save on monthly fees.
Step-by-Step — Create Your 360dialog Account
1
Go to 360dialog.com and click "Get Started"
Open your browser and go to 360dialog.com. On the homepage click the button that says Get Started or Sign Up. A registration form appears asking for your name, email, and password. Fill it in and click Create Account.
Use your business email — not a personal Gmail. This account will be associated with the phone numbers you manage for clients. If you are setting this up for a client, use their business email and create the account in their name.
2
Verify your email
After registering, 360dialog sends a verification email. Open your email inbox, find the email from 360dialog, and click the verification link. You are taken back to the 360dialog dashboard.
3
Add a new WhatsApp number — click "Add Channel"
In the 360dialog dashboard, look for a button that says Add Channel or + New Number — it is usually in the top right or in the left sidebar under "Channels." Click it.
Important: You need a dedicated phone number for the WhatsApp bot — NOT the client's personal number and NOT a number already registered on WhatsApp. Options:

Option A: A new SIM card — buy a cheap prepaid SIM (~$5–$10) and use that number.
Option B: A virtual phone number — services like TextNow, Google Voice (US only), or MySudo give you a real phone number digitally. Some of these work for WhatsApp verification, some don't — test first.

Tell the client: "We need a dedicated phone number for the bot. Buy a cheap prepaid SIM at any phone store for about $5. This becomes the bot's number."
4
Choose your plan and enter the phone number
360dialog asks you to select a plan. Choose the Basic or Starter plan — approximately $5–$10/month. Enter the phone number you want to use for the bot. Enter the country code first — for Romania it is +40, for example +40712345678.
5
Verify the phone number via SMS or call
360dialog sends a verification code to the phone number you entered. Make sure you have access to that SIM card right now — the code expires in a few minutes. Enter the 6-digit code in the verification field and click Verify.
If you are setting this up for a client remotely, ask them to be available to receive the SMS during this step. Have them read you the code over WhatsApp or email.
6
Copy your API Key and Webhook URL
After verification, 360dialog shows you your channel settings. You need two things from this page — copy both and save them somewhere safe:
API Key: A long string of letters and numbers that looks like AbCdEf123456789.... This is your authentication key — Make.com uses it to send messages on behalf of this number.

Webhook URL field: This is where you will paste the Make.com webhook URL in Lesson 2. Leave this field empty for now — you will come back to it.

Save the API Key in your Notes app or Google Keep right now.
⚠️ Important — Number Registration Rules
A phone number can only be registered on ONE WhatsApp account at a time. If the number is currently active on regular WhatsApp or WhatsApp Business app, you must first delete that WhatsApp account from the phone before registering it with 360dialog. Tell your client: "Once we register this number with the API, you cannot use it on your personal WhatsApp app anymore. That is why we use a dedicated SIM."
Key Takeaways — Lesson 1
WhatsApp Business App cannot be automated — you need the WhatsApp Business API through a provider like 360dialog.
Always use a dedicated phone number for the bot — never the client's personal number.
360dialog costs $5–$10/month and is the fastest to set up. Perfect for your first bots.
Save your API Key immediately after creation — you will need it in Lesson 2.
✦ Action — Set Up 360dialog Now
Go to 360dialog.com and create your account. If you have a spare SIM card or a number you can use for testing, register it and copy your API key. If not — just create the account and explore the dashboard so you know exactly where everything is when you do it for a client. The account is free to create, you only pay when you add a number.

Step 1 — Create the Webhook in Make.com
Unlike the automations in Module 2 that use a scheduled trigger, the WhatsApp bot uses a Webhook trigger — it activates instantly every time a message arrives, 24/7.
1
Create a new Scenario in Make.com
Open Make.com. Go to Scenarios. Click + Create a new scenario. The canvas opens with the empty trigger circle.
2
Click the trigger circle and search "Webhooks"
Click the trigger circle. In the search box type Webhooks. Click on the "Webhooks" app that appears — it has a purple icon. From the list of actions select Custom Webhook.
"Custom Webhook" means Make.com will listen at a special URL and trigger the scenario every time data arrives at that URL. 360dialog will send WhatsApp messages to this URL.
3
Click "Add" to create a new webhook and copy the URL
In the Webhooks settings, click Add to create a new webhook. Give it a name like "WhatsApp Bot — [Client Name]". Click Save. Make.com generates a unique webhook URL that looks like:
https://hook.eu1.make.com/abcdef123456789...

Click the Copy address to clipboard button to copy this URL. Save it — you will paste it into 360dialog in the next step.
4
Paste the webhook URL in 360dialog
Go back to 360dialog. Open your channel settings (click on the phone number you registered). Find the field called Webhook URL. Paste the Make.com webhook URL you just copied. Click Save.
This tells 360dialog: "Every time a WhatsApp message arrives on this number, send all the message data to this Make.com URL." From this moment, every incoming WhatsApp message triggers your Make.com scenario.
5
Send a test WhatsApp message to capture the data structure
Go back to Make.com. The Webhooks module should now say "Waiting for data..." with a loading indicator. Send a WhatsApp message to the bot's phone number from your own phone — type anything like "Hello, test message."
Within seconds Make.com receives the message data and the module stops showing "Waiting for data" — it now shows a green checkmark and "1 bundle." Click OK to confirm the data structure.

Make.com has now learned what data 360dialog sends. This data includes: the sender's phone number, the message text, the timestamp, and more. You will use these variables in the next steps.
Step 2 — Add the OpenAI Module
1
Click "+" after the Webhooks module and add OpenAI
Hover over the Webhooks circle and click the + button. Search for OpenAI and select Create a Chat Completion. Connect your OpenAI API key (same one from Module 2).
2
Select Model: gpt-4o
In the Model dropdown select gpt-4o.
3
Set the System message — define the bot's personality
In the System field, paste this template and customize it for the specific client:
You are the AI assistant of [BUSINESS NAME], a [TYPE OF BUSINESS] located in [CITY]. Your job is to reply to customer WhatsApp messages in a friendly, professional and helpful way. About the business: - Name: [BUSINESS NAME] - Services/Products: [LIST MAIN SERVICES] - Hours: [OPENING HOURS] - Address: [ADDRESS] - Phone for urgent matters: [PHONE] Rules: - Always reply in the same language the customer used - Keep replies short — maximum 3 sentences - If the customer asks something you don't know, say: "Great question! Let me check and get back to you shortly." - Always end with a helpful suggestion or question to keep the conversation going - Never make up information about prices, availability or services if you are not sure
Customize every line in [brackets] with the client's real information before going live. The quality of the bot's replies depends entirely on how well this System message is written.
4
Set the User message — the incoming WhatsApp message
In the Message content field, click inside it. Then click the variable picker and find the Webhooks module data. Look for the field that contains the message text — it is usually called messages[] → text → body or similar. Click it to insert the variable.
The exact variable name depends on how 360dialog formats the data. After capturing your test message in Step 1, click on the Webhooks module circle and look at the data it received. You will see a structure like:

messages[].text.body — this is the message text
messages[].from — this is the sender's phone number

Insert messages[].text.body as the User message.
5
Click OK to save the OpenAI module
Scroll down and click OK. The OpenAI circle appears on your canvas connected to the Webhooks circle.
Step 3 — Send the Reply Back via WhatsApp
Now add the module that sends the AI-generated reply back to the customer on WhatsApp. You will use the HTTP module to call the 360dialog API directly.
1
Click "+" and add the HTTP module
Hover over the OpenAI circle and click +. In the search box type HTTP. Click on the HTTP app. Select Make a Request.
The HTTP module lets you call any API endpoint. You will use it to call the 360dialog API to send a WhatsApp message.
2
Configure the HTTP Request settings
Fill in the HTTP module settings exactly as follows:
URL:
https://waba.360dialog.io/v1/messages
Method: Select POST from the dropdown

Headers: Click Add item and add:
— Name: D360-API-KEY | Value: paste your 360dialog API key here
— Name: Content-Type | Value: application/json

Body type: Select Raw

Content type: Select JSON (application/json)
3
Set the Request Body — the message to send
In the Body field, paste this JSON structure. Replace the variable parts using the variable picker:
{ "recipient_type": "individual", "to": "{{1.messages[].from}}", "type": "text", "text": { "body": "{{2.choices[].message.content}}" } }
{{1.messages[].from}} — this is the sender's phone number from the webhook data (Module 1). Click the variable picker to find and insert it.

{{2.choices[].message.content}} — this is the AI-generated reply text from OpenAI (Module 2). Click the variable picker → select OpenAI module → choices[] → message → content.

The phone number format must include the country code without the + sign. For example, a Romanian number +40712345678 becomes 40712345678 in the JSON body.
4
Click OK to save the HTTP module
Click OK. You now have all three modules on your canvas: Webhooks → OpenAI → HTTP (Send WhatsApp).
Step 4 — Test the Complete Bot
1
Turn on the Scenario
Unlike Module 2 where you used "Run once," a webhook-based scenario stays on permanently. Click the Scheduling toggle at the bottom left and turn it ON. The scenario is now live and waiting.
2
Send a real WhatsApp message to the bot number
From your personal phone, open WhatsApp and send a message to the bot's phone number. Send a realistic customer message like: "Hi, what are your opening hours?" or "Do you do deliveries?"
3
Wait 5–10 seconds and check for the reply
Within 5–10 seconds you should receive a WhatsApp reply from the bot number. Read the reply carefully — does it sound natural? Does it match the business information you put in the System message? Is it in the right language?
If you receive no reply after 30 seconds:
1. Open Make.com and check the Scenario history — click the clock icon at the bottom of the editor to see recent runs
2. If the Webhooks module shows no data — the webhook URL in 360dialog was not saved correctly. Go back and re-paste it.
3. If the OpenAI module shows red — check that your API key is correct and that you have credits
4. If the HTTP module shows red — the body JSON has a formatting error. Re-check the JSON structure for missing quotes or brackets
4
Save the Scenario
Click Save in the top right. The bot is now live. Every WhatsApp message that arrives on this number will trigger the scenario and receive an AI reply within seconds.
🎉 Your WhatsApp Bot is Live
You just built a real WhatsApp AI bot. It replies to every message within seconds, 24 hours a day, 7 days a week, in any language the customer uses. This system alone is worth $500–$1,500 to a client plus $150–$300/month recurring. You built it in under 2 hours.
Key Takeaways — Lesson 2
WhatsApp bots use a Webhook trigger — not a scheduled trigger. They activate instantly every time a message arrives.
The System message is the most important part of the bot. The better you describe the business, the better the replies.
The phone number in the JSON body must not have a + sign — use the country code directly (e.g., 40712345678 not +40712345678).
Turn on the Scenario toggle and leave it on permanently — do not use "Run once" for webhook-based automations.
