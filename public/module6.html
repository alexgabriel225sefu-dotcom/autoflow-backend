Setup · Lesson 1

Instagram Graph API — Complete Setup

To automate Instagram DMs you need access to the Instagram Graph API through a Facebook Developer App. This process has more steps than WhatsApp but it is completely free. Follow every step exactly and you will have API access in about 30 minutes.

What You Need Before Starting
Before you begin the setup, make sure these four things are in place. If any of them is missing, the setup will not work:
1
An Instagram Business Account (not Personal)
The client's Instagram account must be set to "Business" — not Personal and not Creator. To check: open Instagram → go to the profile → tap the three lines menu (☰) → Settings → Account → Account type. If it says "Switch to Professional Account" — it is Personal. Tap it and switch to Business.
Why Business account? Only Instagram Business accounts can be connected to the Messenger API which is what we use for DM automation. Personal accounts cannot be automated at all.
2
A Facebook Page connected to the Instagram account
The Instagram Business account must be linked to a Facebook Page. To connect them: open Facebook on desktop → go to your Facebook Page → Settings → Instagram → click "Connect Account" → log in to the Instagram account.
If the client does not have a Facebook Page, they need to create one. Go to facebook.com → click "Create" → "Page" → fill in the business name and category → click Create Page. This takes 2 minutes.
3
Access to the Facebook account that manages the Page
You need to be able to log in to the Facebook account that owns or administers the Facebook Page. During setup you will generate an access token using this account. If you are working for a client, they need to be present or give you temporary access to their Facebook account for this step.
4
A developer account on developers.facebook.com
Go to developers.facebook.com and log in with your Facebook account (not the client's). If it asks you to register as a developer, click Register and confirm. This is free and instant.
Step 1 — Create a Facebook App
1
Go to developers.facebook.com and click "My Apps"
After logging in to developers.facebook.com, look at the top right corner of the page. You see a button or dropdown that says My Apps. Click it. A page shows all your existing apps — it may be empty if this is your first time.
2
Click "Create App"
On the My Apps page, click the green button Create App. A setup wizard opens asking you to choose the type of app you want to build.
3
Select "Business" as the app type
The wizard shows several app type options: Consumer, Business, None of the above. Select Business and click Next.
Why Business? The Business app type gives you access to the Instagram Messaging API and the Pages API — which are exactly what you need for DM automation.
4
Fill in the app details
The next screen asks for:
App name: Type something like "Instagram DM Bot — [Client Name]" or just "DM Automation"

App contact email: Your email address

Business Account: If you have a Meta Business Account, select it. If not, you can skip this for now — click "I don't want to connect a business portfolio yet"

Click Create App. Facebook may ask you to re-enter your password for security. Enter it and click Submit.
5
You are now in the App Dashboard
After creating the app, you land on the App Dashboard. This is where you add products and configure settings. On the left sidebar you see: Dashboard, Settings, Roles, and more. In the main area you see a list of products you can add to your app.
Step 2 — Add Instagram Messaging to Your App
1
Find "Instagram" in the product list and click "Set up"
In the App Dashboard main area, scroll down to find the products list. Look for Instagram — it has the Instagram gradient icon. Click the Set up button next to it.
If you don't see Instagram in the list, look for a button that says "Add Product" or scroll to the bottom of the products section. You can also find it by going to the left sidebar → click the "+" icon next to Products.
2
In the Instagram settings, click "API Setup with Instagram Login"
After adding Instagram, the left sidebar now shows an "Instagram" section. Click on Instagram in the sidebar to expand it. Click on API Setup or Basic Display.
The exact menu names change occasionally as Meta updates the interface. Look for anything related to "API Setup," "Messaging," or "Instagram Basic Display." The goal is to get to the page where you can generate an Access Token.
3
Add the Instagram account as a Tester
On the Instagram API Setup page, look for a section called Instagram Testers or Add Instagram Testers. Click Add Instagram Testers. In the popup, type the Instagram username of the business account you want to connect. Click Submit.
After adding them as a tester, the Instagram account owner needs to accept the tester invitation. To accept:
1. Open Instagram on the phone of the account owner
2. Go to Settings → Apps and Websites → Tester Invites
3. Find your app name and tap Accept

Once accepted, the account is connected to your app as a tester.
Step 3 — Generate the Access Token
1
Go to the Graph API Explorer
Open a new tab and go to developers.facebook.com/tools/explorer. This is Meta's official tool for generating API tokens. Make sure you are logged in with the Facebook account that manages the Page.
2
Select your app from the dropdown
At the top right of the Graph API Explorer you see a dropdown that says "Meta App." Click it and select the app you just created (the "Instagram DM Bot" app).
3
Click "Generate Access Token"
Click the blue button Generate Access Token. A permissions popup appears. You need to add these specific permissions before generating:
Click Add a Permission and add each of these one by one:

✓ instagram_manage_messages — allows reading and sending DMs
✓ pages_messaging — allows messaging through Facebook Pages
✓ instagram_basic — basic Instagram account access
✓ pages_show_list — allows listing connected Pages

After adding all permissions, click Generate Access Token. A Facebook login popup asks you to authorize the app — click Continue as [Your Name] then Done.
4
Copy the Short-Lived Token
The Explorer now shows a long string in the "Access Token" field — this is a short-lived token that expires in about 1–2 hours. Copy it. You will convert it to a long-lived token in the next step.
5
Convert to a Long-Lived Token (valid 60 days)
Open a new browser tab and paste this URL, replacing the placeholders with your real values:
https://graph.facebook.com/v18.0/oauth/access_token?grant_type=fb_exchange_token&client_id=YOUR_APP_ID&client_secret=YOUR_APP_SECRET&fb_exchange_token=YOUR_SHORT_LIVED_TOKEN
Where to find each value:

YOUR_APP_ID: Go to your app dashboard → Settings → Basic → "App ID" is at the top
YOUR_APP_SECRET: Same page → "App Secret" — click "Show" to reveal it
YOUR_SHORT_LIVED_TOKEN: The token you just copied from Graph API Explorer

Press Enter. The browser shows a JSON response like:
{"access_token":"EAAB...long token here...","token_type":"bearer"}

Copy the long access_token value. This is your Long-Lived Token — it lasts 60 days. Save it in your Notes app immediately.
6
Get the Page ID and Instagram Account ID
You need two more values. Go back to Graph API Explorer. In the query field, replace the default text with:
me/accounts
Click Submit. The response shows your Facebook Pages with their IDs. Find the Page connected to the Instagram account and copy its id value — this is your Page ID.

Then query:
{PAGE_ID}?fields=instagram_business_account
Replace {PAGE_ID} with the actual Page ID number. The response shows the Instagram account ID. Copy it — you will need it in Lesson 2.
⚠️ Token Expiry — Set a Reminder
Long-lived tokens expire after 60 days. Set a reminder in your calendar for Day 55 to refresh the token. To refresh: simply call the same long-lived token URL from Step 5 again with the current token. The response gives you a new 60-day token. Add this to your monthly maintenance routine for every Instagram bot client.
Key Takeaways — Lesson 1
Instagram requires a Business Account + connected Facebook Page before any automation is possible.
Always convert the short-lived token to a long-lived token (60 days) — the short one expires in 1–2 hours.
Save your App ID, App Secret, Long-Lived Token, Page ID, and Instagram Account ID in one secure document.
Set a calendar reminder at Day 55 to refresh the token before it expires.
✦ Action — Complete the API Setup
Go through all the steps in this lesson using your own Instagram Business account (or a test account). Create the Facebook App, add Instagram, generate the long-lived token, and save all the values. By the end you should have: App ID, App Secret, Long-Lived Access Token, Page ID, and Instagram Account ID — all saved somewhere safe. You will use all of them in Lesson 2.

Step 1 — Create the Webhook in Make.com
1
Create a new Scenario with a Custom Webhook trigger
In Make.com, click + Create a new scenario. Click the trigger circle. Search Webhooks and select Custom Webhook. Click Add to create a new webhook, name it "Instagram DM Bot — [Client Name]", click Save.
Copy the webhook URL that Make.com generates. It looks like: https://hook.eu1.make.com/abc123.... Save it — you will use it in the next step.
2
Register the Webhook with Meta — Step A: Verification
Go back to your Facebook App dashboard at developers.facebook.com. In the left sidebar, find Webhooks or look under the Messenger/Instagram product. Click Add Callback URL or Edit Callback URL.
Two fields appear:

Callback URL: Paste your Make.com webhook URL here
Verify Token: Type any word you want — for example myverifytoken123. Remember this word — you need it in the next step.

Click Verify and Save. Meta sends a verification request to your Make.com webhook URL. Make.com must respond correctly for the verification to pass.
3
Handle the Webhook Verification in Make.com
The Meta webhook verification sends a GET request with a challenge parameter that Make.com must return. To handle this automatically, before clicking "Verify and Save" in Meta, go to Make.com and click Run once so it is actively listening. Then click Verify and Save in Meta.
Make.com captures the verification request. The webhook module shows data. Click OK.

Note: Meta's webhook verification is more complex than 360dialog. If the verification fails the first time, do not panic. Try this alternative approach:

1. In your Make.com scenario, add a Router module after the Webhook
2. One route handles GET requests (verification) — responds with the challenge value
3. The other route handles POST requests (actual messages)

This advanced setup is covered in Module 11 (Webhook Mastery in the PRO course). For now, try the simple approach first.
4
Subscribe to Instagram Messages
After the callback URL is verified, you need to subscribe to the specific events you want to receive. In the Webhooks section of your Facebook App, find Instagram in the list of subscription options. Check the box for messages. Click Subscribe.
This tells Meta: "Every time a DM arrives on the connected Instagram account, send the data to my Make.com webhook URL." You are now subscribed to message events.
Step 2 — Add the OpenAI Module
1
Click "+" and add OpenAI → Create a Chat Completion
In Make.com, hover over the Webhook circle and click +. Search OpenAI and select Create a Chat Completion. Connect your OpenAI API key. Select model gpt-4o.
2
Write the System message for the Instagram bot
The Instagram bot personality should be slightly more casual than the WhatsApp bot — Instagram is a social platform. Paste this in the System field and customize:
You are the friendly AI assistant of [BUSINESS NAME] on Instagram. Your job is to reply to Instagram direct messages in a warm, casual and helpful tone that matches the brand's Instagram voice. About the business: - Name: [BUSINESS NAME] - What we sell/offer: [PRODUCTS OR SERVICES] - Price range: [PRICE RANGE] - Location/Delivery: [LOCATION OR DELIVERY INFO] - How to order/book: [ORDERING PROCESS] - Instagram handle: @[HANDLE] Rules: - Keep replies short — 1 to 3 sentences maximum - Use a friendly, natural tone — like a real person on Instagram - If asked about prices, give the range and invite them to DM for a custom quote - If asked about availability, say you will check and get back to them - Always end with a question or suggestion to keep the conversation going - Never make up specific prices or dates - Reply in the same language the person used
3
Set the User message using the DM text from the webhook
In the Message content field, use the variable picker to find the message text from the webhook data. After capturing a test DM, the webhook data structure from Meta looks like this:
The message text is usually at: entry[].messaging[].message.text

Click the variable picker → select the Webhooks module → navigate to entry → messaging → message → text

The sender's ID (you need this to send the reply) is at: entry[].messaging[].sender.id
4
Capture a real test DM to confirm the data structure
The easiest way to find the exact variable paths is to capture a real message first. Turn on your Make.com scenario (Scheduling ON). Send a DM to the Instagram account from your own Instagram account. Make.com receives the data. Click on the Webhook module to see the exact data structure and find the correct variable paths.
Step 3 — Send the Reply via Instagram
1
Add the HTTP module to send the Instagram reply
Hover over the OpenAI circle and click +. Search HTTP and select Make a Request.
2
Configure the HTTP module for Instagram Messaging API
Fill in the HTTP module settings:
URL:
https://graph.facebook.com/v18.0/me/messages
Method: POST

Query String — add these parameters:
— Name: access_token | Value: paste your Long-Lived Access Token here

Body type: Raw
Content type: JSON (application/json)
3
Set the Request Body
In the Body field paste this JSON. Replace the variable parts using the variable picker:
{ "recipient": { "id": "{{1.entry[].messaging[].sender.id}}" }, "message": { "text": "{{2.choices[].message.content}}" }, "messaging_type": "RESPONSE" }
{{1.entry[].messaging[].sender.id}} — the sender's Instagram ID from the webhook (Module 1). Use the variable picker to find the exact path after capturing your test message.

{{2.choices[].message.content}} — the GPT-4o generated reply (Module 2). Variable picker → OpenAI module → choices[] → message → content.
4
Test the complete bot
Make sure the Scenario is ON (Scheduling toggle ON). Send a DM to the Instagram account from your own personal Instagram account. Wait 5–10 seconds. You should receive an AI-generated reply in your Instagram DMs.
Common errors and fixes:

"OAuthException: Invalid OAuth access token" — your Long-Lived Token has expired or is incorrect. Generate a new one following the steps in Lesson 1.

"The recipient is not subscribed to this app" — the Instagram account needs to accept the tester invitation in Instagram Settings → Apps and Websites. Or your app needs to go through Meta's App Review for messaging permissions.

No response received at all — the webhook subscription failed. Go back to your Facebook App → Webhooks → check that Instagram messages is subscribed and the callback URL is verified (shows a green checkmark).
🎉 Your Instagram DM Bot is Live
Every Instagram DM sent to the business account now receives an AI-generated reply within seconds. The bot replies in the customer's language, matches the brand voice, and keeps the conversation going. This system is worth $600–$1,200 setup plus $200–$350/month to a client.

Advanced Feature 1 — Comment Auto-Reply
When someone comments on a post asking a question, the bot can reply automatically in the comments. This is extremely valuable for businesses that post product photos and get flooded with "price?" and "available?" comments.
1
Subscribe to Instagram Comments in your Facebook App
Go back to your Facebook App → Webhooks. Find the Instagram subscription. Click Edit and add comments to the subscribed fields. Click Update.
Now Meta sends data to your Make.com webhook not just for DMs but also for new comments on the Instagram account's posts.
2
Add a Router in Make.com to separate DMs from Comments
Since the same webhook now receives both DMs and Comments, you need a Router to send them to different paths. In Make.com, click the + after the Webhook module. Instead of adding a regular module, look for Router in the search — it is a built-in Make.com tool that splits the flow into multiple paths.
After adding the Router, you will see two (or more) paths. Configure them:

Path 1 — DMs: Add a Filter → check if the data contains entry[].messaging — if yes, this is a DM. Connect the OpenAI + HTTP modules from Lesson 2 to this path.

Path 2 — Comments: Add a Filter → check if the data contains entry[].changes[].value.comments — if yes, this is a comment. Add separate OpenAI and HTTP modules for comment replies to this path.
3
Configure the Comment Reply HTTP module
To reply to a comment, use a different API endpoint. The HTTP module for comment replies uses:
https://graph.facebook.com/v18.0/{COMMENT_ID}/replies
Method: POST
Query String: access_token = your Long-Lived Token
Body (JSON):
{ "message": "{{openai_reply}}" }
Where {COMMENT_ID} in the URL is replaced with the variable from the webhook data that contains the comment ID, and {{openai_reply}} is the GPT-4o generated response.
4
Add a Keyword Filter for Comment Replies
Not every comment needs an automated reply. Add a Filter before the OpenAI module in the Comments path that only processes comments containing specific keywords.
In Make.com, click the small wrench icon on the connection line between Router and OpenAI (in the comments path). A Filter window opens. Set the condition:

Field: comment text
Operator: Contains one of
Values: price, pret, cost, available, disponibil, how much, cat costa, order, comand, where, unde, delivery, livrare

This way the bot only replies to comments asking about prices, availability, or ordering — the comments that matter most for business. Normal compliments and general comments are left for the business owner to handle personally.
